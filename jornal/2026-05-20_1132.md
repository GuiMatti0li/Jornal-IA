## Visão Geral


O dia foi dominado por lançamentos substanciais do Google I/O 2026, sinalizando uma aposta clara da gigante de tecnologia na era dos agentes de IA, que transcendem chatbots para executar tarefas complexas e proativas em todo o seu ecossistema de produtos. Em paralelo, o campo da cibersegurança enfrentou uma avalanche de ameaças de cadeia de suprimentos e zero-days, destacando uma escalada na sofisticação dos atacantes impulsionada, em parte, pela própria IA. O lançamento do **Gemini 3.5 Flash** e a confirmação do hack de **3.800 repositórios internos do GitHub** via extensão maliciosa de VS Code foram os movimentos mais significativos, revelando avanços e fragilidades críticas da indústria.

Implicação prática: Desenvolvedores e empresas devem priorizar a integração de IA agentic para otimizar fluxos de trabalho, mas devem redobrar os esforços em segurança de cadeia de suprimentos e gerenciamento de vulnerabilidades, pois a mesma tecnologia que impulsiona a inovação também está elevando o patamar das ameaças.

**Exploits para LPE no kernel Linux: PinTheft e DirtyDecrypt**
Novos exploits de escalada de privilégios local (LPE) foram lançados para vulnerabilidades no kernel Linux. O PinTheft, que explora um double-free no RDS (Reliable Datagram Sockets) em sistemas Arch Linux, permite que atacantes locais obtenham privilégios de root, com PoC público. Similarmente, um PoC para DirtyDecrypt (CVE-2026-31635), uma falha que permite escrita na pagecache devido à falta de COW (copy-on-write) em `rxgk_decrypt_skb()`, também foi lançado e afeta distribuições como Fedora e openSUSE Tumbleweed, permitindo LPE ao modificar arquivos privilegiados.

**Falha de segurança crítica em ChromaDB para aplicativos de IA permite sequestro de servidor**
Uma vulnerabilidade de gravidade máxima (CVE-2026-45829) na versão mais recente do Python FastAPI do ChromaDB, um banco de dados vetorial open-source para aplicações de IA, permite que atacantes não autenticados executem código arbitrário em servidores expostos. A falha reside em um endpoint de API que permite a incorporação de configurações de modelo antes da verificação de autenticação, forçando o ChromaDB a carregar e executar um modelo malicioso do Hugging Face. A autenticação só ocorre após a execução do modelo, permitindo que um atacante obtenha acesso de shell completo ao processo do servidor. Aproximadamente 73% das instâncias expostas na internet estão executando uma versão vulnerável. A HiddenLayer, que descobriu a falha, tentou contatar o desenvolvedor várias vezes sem resposta. Recomenda-se restringir o acesso à rede à porta da API do ChromaDB ou usar o frontend Rust.
[Bleeping Computer](https://www.bleepingcomputer.com/news/security/max-severity-flaw-in-chromadb-for-ai-apps-allows-server-hijacking/) / [SecurityWeek](https://www.securityweek.com/unpatched-chromadb-vulnerability-can-lead-to-server-takeover/)

**Novas zero-days YellowKey e GreenPlasma para Windows**
Um pesquisador de segurança revelou as vulnerabilidades YellowKey (CVE-2026-45585), que permite ignorar o BitLocker, e GreenPlasma, uma escalada de privilégios para SYSTEM no Windows, além de um re-exploit para uma falha antiga.
YellowKey (CVE-2026-45585) permite que um atacante com acesso físico e um drive USB especial ignore a criptografia do BitLocker e obtenha acesso irrestrito a dados, explorando o ambiente de recuperação do Windows (WinRE) sem credenciais. A Microsoft lançou mitigações, incluindo a remoção de `autofstx.exe` do BootExecute e a configuração do BitLocker para o modo "TPM+PIN". GreenPlasma é uma falha de escalada de privilégios local (LPE) que afeta Windows 10, 11 e Server, explorando um componente de gerenciamento de serviços de entrada de texto para obter privilégios SYSTEM. Um terceiro exploit, MiniPlasma, re-explora uma vulnerabilidade já corrigida em 2020 (CVE-2020-17103) no Windows Cloud Files Mini Filter Driver, demonstrando que o patch original foi ineficaz.
[Dark Reading](https://www.darkreading.com/cyberattacks-data-breaches/windows-zero-day-barrage-continues-after-patch-tuesday) / [The Hacker News](https://thehackernews.com/2026/05/microsoft-releases-mitigation-for-yellowkey-bitlocker-bypass-cve-2026-45585-exploit) / [Bleeping Computer](https://www.bleepingcomputer.com/news/microsoft/microsoft-shares-mitigation-for-yellowkey-windows-zero-day/)
