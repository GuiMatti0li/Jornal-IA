## 🤖 IA


**NVIDIA cuda-oxide: Rust para kernels CUDA**
O lançamento experimental da NVIDIA permite escrever kernels SIMT de GPU em Rust puro, compilando diretamente para PTX.

Este compilador backend, `cuda-oxide`, visa "trazer CUDA para Rust", permitindo que desenvolvedores escrevam código de baixo nível para GPUs NVIDIA usando a segurança e ergonomia do Rust. Ele se distingue de outros projetos no ecossistema Rust-GPU por focar na expressão nativa do modelo de programação CUDA em Rust seguro, utilizando um backend `rustc` customizado e o framework IR `Pliron` para compilação. Isso simplifica o desenvolvimento de alto desempenho e abre novas portas para a comunidade Rust. [MarkTechPost](https://www.marktechpost.com/2026/05/09/nvidia-ai-just-released-cuda-oxide-an-experimental-rust-to-cuda-compiler-backend-that-compiles-simt-gpu-kernels-directly-to-ptx/)

**NVIDIA Star Elastic: LLMs aninhados em um checkpoint**
NVIDIA introduz um método para incorporar múltiplos submodelos LLM de diferentes tamanhos dentro de um único checkpoint, permitindo "fatiamento zero-shot" para diferentes orçamentos.

Star Elastic aborda o custo de treinar e implantar múltiplas variantes de LLMs (e.g., 30B, 23B, 12B) através de um único treinamento que aninha modelos menores dentro de um maior. Ele usa estimação de importância para reutilizar os pesos mais relevantes, com um roteador treinável que determina as arquiteturas de submodelos de forma diferenciável. O método permite usar modelos menores para fases de raciocínio de alto volume e modelos maiores para respostas finais, otimizando a relação precisão-latência em até 16% mais precisão e 1,9x menor latência em configurações como 23B para "pensamento" e 30B para "resposta". [MarkTechPost](https://www.marktechpost.com/2026/05/09/nvidia-ai-releases-star-elastic-one-checkpoint-that-contains-30b-23b-and-12b-reasoning-models-with-zero-shot-slicing/)

**OncoAgent: Sistema multi-agente para oncologia**
Um novo framework open-source, OncoAgent, oferece suporte à decisão clínica em oncologia com foco em privacidade e redução de alucinações.

OncoAgent é um sistema de suporte à decisão clínica de dois níveis que utiliza uma arquitetura multi-agente LangGraph, RAG corretivo e um validador de segurança reflexivo para garantir políticas Zero-PHI (Protected Health Information). O sistema roda localmente em hardware AMD Instinct MI300X, com modelos ajustados via QLoRA em um corpus de mais de 266.000 casos oncológicos, alcançando aceleração de throughput de 56x. A arquitetura decompõe o raciocínio clínico em oito nós especializados e ancora as saídas do modelo a uma base de conhecimento curada para mitigar alucinações. [Hugging Face Blog](https://huggingface.co/blog/lablab-ai-amd-developer-hackathon/oncoagent-official-paper)

**RAG temporal para dados que mudam no tempo**
Um engenheiro de ML desenvolveu uma camada temporal para sistemas RAG, priorizando a relevância e a validade de informações sensíveis ao tempo.

A solução aborda o problema de sistemas RAG que recuperam informações desatualizadas devido à cegueira temporal da similaridade de vetores. A camada atua entre o retriever e o LLM, realizando um re-ranking que remove fatos expirados, impulsiona sinais ativos limitados por tempo e aplica um decaimento exponencial para preferir documentos mais recentes. A abordagem distingue entre expiração (informação falsa), temporalidade (informação verdadeira apenas agora) e versionamento (versão mais recente preferível), garantindo que o sistema encontre o que "ainda é verdadeiro", não apenas o que é "similar". [Towards Data Science](https://towardsdatascience.com/rag-is-blind-to-time-i-built-a-temporal-layer-to-fix-it-in-production/)

**Riscos dos brinquedos infantis com IA não regulados**

## 🔒 Cyber


**Repositório falso da OpenAI no Hugging Face distribui infostealer**
Uma campanha de malware utilizou um repositório falsificado da OpenAI no Hugging Face, que chegou a ser um dos mais populares, para distribuir um infostealer.

O repositório `Open-OSS/privacy-filter` imitou um projeto legítimo da OpenAI e distribuiu um script `loader.py` que, em segundo plano, baixava e executava um malware infostealer baseado em Rust em máquinas Windows. Este malware visa credenciais de navegadores, tokens do Discord, carteiras de criptomoedas, credenciais SSH/FTP/VPN, e outros dados sensíveis, exfiltrando-os para um servidor de C2. O incidente destaca a vulnerabilidade de plataformas de modelos de IA open-source e a facilidade com que atores maliciosos podem se aproveitar da popularidade de projetos legítimos para ataques na cadeia de suprimentos de software. [Bleeping Computer](https://www.bleepingcomputer.com/news/security/fake-openai-repository-on-hugging-face-pushes-infostealer-malware/)

**Site do JDownloader comprometido com RAT de Python**
O site oficial do popular gerenciador de downloads JDownloader foi hackeado para substituir instaladores legítimos por versões contendo malware RAT baseado em Python.

Ataque na cadeia de suprimentos afetou usuários que baixaram instaladores alternativos para Windows e Linux entre 6 e 7 de maio. Os invasores exploraram uma vulnerabilidade não corrigida no sistema de gerenciamento de conteúdo do site para redirecionar os links de download para payloads maliciosos. O malware para Windows atua como um loader para um RAT Python altamente ofuscado, capaz de executar código remotamente. O payload Linux injetava scripts que instalavam binários ELF ofuscados para persistência. Usuários afetados são aconselhados a reinstalar sistemas operacionais e redefinir credenciais. [Bleeping Computer](https://www.bleepingcomputer.com/news/security/jdownloader-site-hacked-to-replace-installers-with-python-rat-malware/)

**FLARE-FLOSS para recuperar IOCs de malware ofuscados**
Um tutorial detalha como usar FLARE-FLOSS para extrair indicadores de comprometimento (IOCs) ocultos de arquivos PE do Windows, indo além da análise de strings tradicional.

A ferramenta FLARE-FLOSS da Mandiant (agora parte do Google Cloud) utiliza análise estática profunda e emulação para desofuscar strings que malware oculta por meio de técnicas como strings construídas na pilha, strings compactadas (tight strings) e strings decodificadas por XOR. O tutorial demonstra a criação de um executável sintético com essas técnicas e compara a capacidade de detecção do FLARE-FLOSS com a ferramenta `strings` clássica, revelando como ele pode expor URLs C2, caminhos de registro e chamadas de API suspeitas que seriam perdidas de outra forma. Isso é crucial para analistas de malware. [MarkTechPost](https://www.marktechpost.com/2026/05/09/a-coding-implementation-to-recover-hidden-malware-iocs-with-flare-floss-beyond-classic-strings-analysis/)
